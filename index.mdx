Jami Harness (`@jami-studio/harness`) is the provider-agnostic agent SDK and governed action runtime built for developers who need power without chaos. It gives every agent run clear contracts, policy-gated tool access, durable artifact trails, readable traces, and replaceable module ports — so you can build, inspect, govern, and improve real software work without being locked into any single provider or infrastructure choice.

## Get started in three steps

<Steps>
  <Step title="Install">
    Add the SDK and CLI to your project with your preferred package manager.

    ```bash
    npm install @jami-studio/harness-sdk @jami-studio/harness-cli
    ```
  </Step>
  <Step title="Initialize">
    Run `jami init` inside your project directory to create the local harness configuration.

    ```bash
    jami init --json
    ```
  </Step>
  <Step title="Run your first agent">
    Execute your first governed run. The harness applies policy, records evidence, and exports a durable artifact — all locally.

    ```js
    import { createHarness } from '@jami-studio/harness-sdk';

    const harness = createHarness();
    const result = await harness.run({ runId: 'run_local_example' });

    console.log(result.evidence.evidenceId);
    ```
  </Step>
</Steps>

## Key capabilities

<CardGroup cols={3}>
  <Card title="Governed by default" icon="shield-check">
    Every tool call passes through a default-deny policy engine. Actions require explicit approval before they execute, and every decision is recorded as audit evidence.
  </Card>
  <Card title="Replaceable module ports" icon="puzzle-piece">
    Swap out memory, context, search, storage, provider, policy, tools, artifacts, or observability without changing your run grammar or losing evidence continuity.
  </Card>
  <Card title="Durable artifact trails" icon="box-archive">
    Every run produces a structured evidence packet: artifact records, trace events, approval decisions, and provenance metadata — all tied to a single `evidenceId`.
  </Card>
</CardGroup>
